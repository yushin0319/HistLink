# 【最重要】勝手に進めるな

## 絶対に守るべきルール

**ユーザーの明確な許可がない限り、勝手に作業を進めてはいけない**

### 具体例

❌ **悪い例:**
```
Claude: 「変更しますか？」
（ユーザーの返答を待たずにファイル編集を開始）
```

✅ **良い例:**
```
Claude: 「変更しますか？」
User: 「うん」「はい」「やって」など明確な許可
Claude: （許可を得てから作業開始）
```

## 特に注意が必要な操作

- 設定ファイルの変更（.mcp.json, CLAUDE.md等）
- ファイルの削除
- 重要なコードの変更
- 外部サービスの設定変更
- インストール/アンインストール操作

## 発生した問題（2025-11-13）

Notion MCP設定の変更を提案した後、ユーザーの許可を得ずに`.mcp.json`を読み込んで変更しようとした。

ユーザーから強く指摘：「やれっていったか？」「絶対に忘れんとこに書いとけ　勝手に進めるな」

## 心得

- 「〜しますか？」と聞いたら、必ず返答を待つ
- 「はい」「うん」「やって」などの明確な許可を確認
- 沈黙や曖昧な反応では進めない
- 重要な操作ほど慎重に確認する
